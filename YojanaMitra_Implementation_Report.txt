# YojanaMitra - Project Implementation Report

## Executive Summary

**YojanaMitra** is an AI-powered government welfare scheme discovery platform that helps Indian citizens find and apply for government schemes they are eligible for. The platform uses **Gemini AI** for intelligent scheme matching and provides a bilingual (English/Kannada) interface for accessibility.

**Project Type:** Full-stack web application  
**Primary Language:** Python (Flask backend), JavaScript (Frontend)  
**Database:** SQLite (via SQLAlchemy)  
**AI Integration:** Google Gemini API

## 4. Main Functions Explained

### 4.1 AI/ML Functions

#### `calculate_match_score(user, scheme)`
**Location:** `app.py` (lines 643-783)

**Purpose:** Calculates how well a user matches a scheme's eligibility criteria

**Algorithm:**
```python
def calculate_match_score(user, scheme):
    score = 100.0  # Perfect score
    reasons = []
    
    # Age check (25% weight)
    if scheme.min_age and user.age < scheme.min_age:
        score -= 25
        reasons.append(f"Age below minimum ({scheme.min_age})")
    
    # Income check (20% weight)
    if scheme.max_income and user.income > scheme.max_income:
        score -= 20
        reasons.append("Income exceeds limit")
    
    # Gender check (15% weight)
    # State check (15% weight)
    # Occupation check (10% weight)
    # Caste check (10% weight)
    # Marital status check (5% weight)
    
    return max(score, 0), reasons
```

**Scoring Breakdown:**
- **Age:** 25 points (most critical)
- **Income:** 20 points
- **Gender:** 15 points
- **State/Location:** 15 points  
- **Occupation:** 10 points
- **Caste:** 10 points
- **Marital Status:** 5 points

**Return Value:** `(score: float, reasons: list)`

### 4.2 Core Backend Functions

#### `check_eligibility()` - Anonymous Scheme Matching
**Location:** `app.py` (lines 498-563)

**Purpose:** Allows users to check eligibility WITHOUT creating an account

**Process:**
1. Creates temporary `TempUser` object from form data
2. Fetches all schemes from database
3. Runs `calculate_match_score()` for each scheme
4. Filters schemes with score ≥ 50%
5. Returns sorted list (highest score first)

**Response Format:**
```json
{
  "schemes": [
    {
      "id": 1,
      "name": "PM-KISAN",
      "description": "...",
      "match_score": 95.0,
      "reasons": ["Perfect match!"]
    }
  ]
}
```

---

#### `get_recommendations()` - Logged-in User Recommendations
**Location:** `app.py` (lines 478-495)

**Purpose:** Get personalized recommendations for logged-in users

**Difference from check_eligibility:**
- Requires authentication
- Uses saved user profile from database
- More persistent and personalized

## 7. Key Implementation Details

### 7.1 Session Management
- **Technology:** Flask server-side sessions
- **Secret Key:** Stored in `.env` file
- **Storage:** Server memory (not persistent across restarts)
- **User Session:** `session['user_id']`
- **Admin Session:** `session['admin_id']`

### 7.2 Password Security
- **Hashing:** BCrypt with automatic salt generation
- **Verification:** `bcrypt.checkpw()`
- **Storage:** Never stores plaintext passwords
- **Field:** `password_hash` (200 characters)

### 7.3 CORS Configuration
```python
CORS(app, 
     origins=["http://localhost:5000"],
     supports_credentials=True)
```
- Allows cross-origin requests
- Supports authentication cookies
- Limited to localhost for security

### 7.4 Database Initialization
**Function:** `init_db()`

**Process:**
1. Creates all tables
2. Checks for existing admin
3. Creates default admin if none exists
   - Email: `admin@yojana.com`
   - Password: `admin123`

**Execution:** Runs automatically on app start

## 8. Data Flow Examples

### 8.1 Eligibility Check Flow

```
1. User fills 60+ field form → POST /api/check-eligibility
                                        ↓
2. Backend creates TempUser object with form data
                                        ↓
3. Fetches all schemes from database (Scheme.query.all())
                                        ↓
4. For each scheme: calculate_match_score(user, scheme)
                                        ↓
5. Filter schemes with score ≥ 50
                                        ↓
6. Sort by score (descending)
                                        ↓
7. Return JSON with schemes + match_scores
                                        ↓
8. Frontend displays cards with scores
```

### 8.2 Admin Scheme Creation Flow

```
1. Admin logs in → Session created (admin_id)
                            ↓
2. Clicks "Add Scheme" → Renders complex form
                            ↓
3. Fills form with 20+ fields (including multi-select dropdowns)
                            ↓
4. Submit → POST /api/schemes with JSON payload
                            ↓
5. Backend validates admin session
                            ↓
6. Converts array fields to JSON strings
                            ↓
7. Creates Scheme object
                            ↓
8. Saves to database (db.session.add + commit)
                            ↓
9. Returns success + new scheme ID
                            ↓
10. Frontend refreshes schemes table
```
